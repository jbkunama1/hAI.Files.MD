hAI.Files.MD

Self-hosted files.md Instanz

Deine privaten Markdown-Notizen

Privat. Lokal. Self-hosted. Synchronisiert über deinen eigenen Server und Telegram-Bot.

📖 Dokumentation (DE) 📖 Documentation (EN) GitHub Repo

⚡ Quickstart

git clone https://github.com/jbkunama1/hAI.Files.MD.git
cd hAI.Files.MD
git clone https://github.com/zakirullin/files.md /opt/filesmd/files.md
export TOKENS_SALT="$(head -c 32 /dev/urandom | base64)"
export BOT_API_TOKEN="dein_telegram_bot_token"
docker compose up -d

🤖 Telegram-Bot einrichten Pflicht für Sync-Login

Der Sync-Login läuft ausschließlich über einen Telegram-Bot. Ohne Bot ist kein Geräte-Login möglich.

1
Bot erstellen via BotFather

Telegram öffnen → @BotFather öffnen → /newbot eingeben → Name & Username vergeben → API-Token kopieren

2
Token in docker-compose.yml eintragen

BOT_API_TOKEN=123456789:AAFxxxxxxx als Umgebungsvariable setzen

3
Container neu starten

docker compose down && docker compose up -d

4
Gerät verknüpfen

Im Telegram-Bot /app eingeben → Link öffnen → Gerät ist verknüpft ✅

⚠️ Wichtig: APP_URL in der docker-compose.yml muss exakt mit der URL übereinstimmen, von der du die PWA aufrufst — inkl. https:// und ohne Port.

🔄 Sync-Methoden

📁 Local-first

Kein Sync. Dateien bleiben auf dem Gerät. Maximale Privatsphäre.

☁️ Cloud-Ordner

iCloud, Dropbox oder Google Drive als gemeinsamer Ordner. Kein eigener Server nötig.

🖥️ Self-hosted diese Instanz

Eigener Server mit Go-Binary. Volle Kontrolle über Daten.

🌐 Hosted

api.files.md — öffentlicher Server, sofort nutzbar ohne Setup.

⚙️ Wichtige Umgebungsvariablen

🔑
BOT_API_TOKEN

Telegram Bot Token von BotFather. Pflicht für Sync-Login.

🌐
APP_URL / API_URL

Exakte URL der PWA inkl. https://. Muss mit Browser-Origin übereinstimmen (CORS).

🔒
TOKENS_SALT

Zufälliger Schlüssel zum Signieren von Auth-Tokens. Einmal generieren, sicher aufbewahren.

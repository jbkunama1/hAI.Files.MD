hAI.Files.MD

Self-hosted Docker stack for files.md – PWA frontend & sync server in one container.

Language / Sprache: 🇩🇪 Deutsch · 🇬🇧 English

🛠️ Quick start

sudo mkdir -p /opt/filesmd && cd /opt/filesmd
git clone https://github.com/zakirullin/files.md.git
git clone https://github.com/jbkunama1/hAI.Files.MD.git
mkdir -p hAI.Files.MD/app/storage hAI.Files.MD/app/tokens

cd hAI.Files.MD
# TOKENS_SALT in docker-compose.yml eintragen
docker compose build --no-cache
docker compose up -d
💡 TOKENS_SALT generieren: head -c 32 /dev/urandom | base64

🔄 Sync – gleiche MDs auf allen Geräten

Einmalig pro Gerät in der DevTools-Konsole (F12 → Console) ausführen:

localStorage.setItem('ApiHost', 'http://DEIN-SERVER:PORT');

Ab sofort synct die PWA automatisch. Alle Geräte, die auf denselben Server zeigen, teilen dieselben Markdown-Dateien.

Handy  ──┬
PC     ──┼──►  http://SERVER:PORT  ──►  /app/storage
Tablet ──┘
⚠️ Ohne API_URL in der docker-compose.yml startet die Sync-API nicht!

💻📱🍎 Zugriff je Gerät

💻 Laptop

Browser öffnen → http://DEIN-SERVER:PORT, dann mit F12 die Konsole öffnen und einmalig ausführen:

localStorage.setItem('ApiHost', 'http://DEIN-SERVER:PORT');

Optional in Chrome oder Edge als App installieren.

📱 Android

Chrome öffnen, Seite aufrufen, dann „Zum Startbildschirm hinzufügen“. Danach in der Adressleiste:

javascript:localStorage.setItem('ApiHost','http://DEIN-SERVER:PORT');
⚠️ Manche Android-Browser entfernen javascript: beim Einfügen. Dann zuerst javascript: tippen und den Rest ergänzen.

🍎 iPad

Safari öffnen, Seite aufrufen, dann über den Teilen-Button „Zum Home-Bildschirm“ hinzufügen. ApiHost per Bookmarklet setzen:

javascript:localStorage.setItem('ApiHost','http://DEIN-SERVER:PORT');

✅ Sync prüfen

localStorage.getItem('ApiHost');

Die Ausgabe sollte http://DEIN-SERVER:PORT sein.

🐛 Troubleshooting

❌ path not found (Portainer)

Repos zuerst klonen, dann absoluten Pfad in docker-compose.yml setzen:

build:
  context: /opt/filesmd/files.md
  dockerfile: /opt/filesmd/files.md/Dockerfile

❌ Sync funktioniert nicht

Prüfen ob API_URL gesetzt ist:

docker inspect files-md | grep API_URL

❌ Timeout / kein Internet aus Docker

curl -I https://github.com
docker run --rm alpine sh -c "nslookup proxy.golang.org"

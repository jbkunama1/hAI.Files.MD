hAI.Files.MD

Self-hosted Docker stack for files.md – PWA frontend & sync server in one container.

Language / Sprache: 🇩🇪 Deutsch · 🇬🇧 English

🛠️ Quick start

sudo mkdir -p /opt/filesmd && cd /opt/filesmd
git clone https://github.com/zakirullin/files.md.git
git clone https://github.com/jbkunama1/hAI.Files.MD.git
mkdir -p hAI.Files.MD/app/storage hAI.Files.MD/app/tokens

cd hAI.Files.MD
# Set TOKENS_SALT in docker-compose.yml
docker compose build --no-cache
docker compose up -d
💡 Generate TOKENS_SALT: head -c 32 /dev/urandom | base64

⚙️ Environment variables

VariableRequiredDescription
APP_URLrecommendedURL of the web app
API_URLfor syncEnables sync server – same URL as APP_URL is fine
STORAGE_DIRyesNotes storage path
TOKENS_DIRyesAuth tokens path
TOKENS_SALTrecommendedRandom value for token signing
CERT_DIRnoTLS certificate path (empty = HTTP only)
LOG_FILEnoLog file path
STORAGE_QUOTA_KBnoStorage limit per user in KB (default 1024)
BOT_API_TOKENnoTelegram bot token (optional)
⚠️ Without API_URL the sync server will not start!

🐛 Troubleshooting

❌ path not found (Portainer)

Clone both repos first, then use absolute paths in docker-compose.yml:

build:
  context: /opt/filesmd/files.md
  dockerfile: /opt/filesmd/files.md/Dockerfile

❌ Sync not working

Check that API_URL is set:

docker inspect files-md | grep API_URL

❌ Timeout / no internet from Docker

curl -I https://github.com
docker run --rm alpine sh -c "nslookup proxy.golang.org"

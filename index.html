hAI.Files.MD

Self-hosted Docker stack for files.md – with PWA frontend and sync server.

Language / Sprache: 🇩🇪 Deutsch · 🇬🇧 English

Quick start

Clone both repos on your server, then deploy the stack with Docker Compose or Portainer.

sudo mkdir -p /opt/filesmd && cd /opt/filesmd
git clone https://github.com/zakirullin/files.md.git
git clone https://github.com/jbkunama1/hAI.Files.MD.git
mkdir -p hAI.Files.MD/data hAI.Files.MD/app/storage

cd hAI.Files.MD
# Edit docker-compose.yml: set SALT and HOST
docker compose build
docker compose up -d

Open the PWA at http://YOUR-SERVER:8080 and point it to the sync server at http://YOUR-SERVER:3333 via localStorage.setItem('ApiHost', 'http://YOUR-SERVER:3333') in the browser console.

🐛 Troubleshooting

❌ path not found (Portainer)

Portainer cannot find the build context. Clone both repos first (see Quick start above), then use absolute paths in docker-compose.yml:

files-md-sync:
  build:
    context: /opt/filesmd
    dockerfile: /opt/filesmd/hAI.Files.MD/Dockerfile.sync

❌ Timeout during build

The default Dockerfile.sync clones the upstream repo during build. If your server has no outbound internet access this will time out.

Fix: Use the local build variant – copy from the already-cloned host folder instead:

# In Dockerfile.sync replace the git clone lines with:
COPY files.md/sync ./sync
⚠️ context: /opt/filesmd must point to the parent folder containing both repos, so the COPY files.md/sync path resolves correctly.

❌ Test internet access from Docker

curl -I https://github.com
docker run --rm alpine sh -c "apk add git && git clone https://github.com/zakirullin/files.md.git /tmp/test"
